# INC — Ideate < Narrate > Create

This folder is the working context layer of Lincoln Mitchell's portfolio. It's not documentation — it's the accumulated knowledge that agents and humans draw from to ideate, narrate, and create.

```
inc/
├── ideate/       ← NorthStar exploration, concepts, divergent thinking
├── narrate/      ← Structured context, domain knowledge, positioning
│   └── ids-2026/ ← 16 IDS conference talk analyses
└── create/       ← Implementation decisions, shipped artefacts, patterns
```

## The Framework

- **I (Ideate)** — Discovery-first NorthStar prototyping. What could exist.
- **<N> (Narrate)** — The agentic design system infrastructure that amplifies both directions. Domain knowledge encoded as context any agent can consume.
- **C (Create)** — Something worth showing. Each cycle produces a deliverable at increasing fidelity:

| Stage | C Deliverable | Scale |
|---|---|---|
| 1 | NorthStar Prototype | 1 person + agent |
| 2 | Proof of Concept | 1-2 people |
| 3 | Working Prototype | Small team |
| 4 | MVP | Cross-functional team |
| 5 | Beta | Full team |
| 6 | V1 | Full team + ops |
| 7 | Product | Organisation |

Every stage is a full I<N>C cycle. The NorthStar feeds back into N, which reshapes I for the next cycle. The MVP feeds usage data back into N, which reshapes I for V2. Each C becomes context for the next I.

N is the expansion layer — not a step between I and C, but the amplifier that radiates into both: `I <N> C`

**Everyone contributes to N. Anyone can draw from it.**

## Why This Exists in the Repo

This folder practices what it describes. The IDS 2026 talk analyses in `narrate/` were extracted from FigJam via Figma MCP, structured by Claude Code agents, and positioned against Lincoln's methodology — all using the same agentic workflow the site demonstrates.

The folder structure IS the methodology.
