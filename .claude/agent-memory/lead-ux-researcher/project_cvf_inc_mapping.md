Quinn & Cameron's Competing Values Framework (1983) maps directly to INC:

- **I (Ideate)** = Adhocracy — external focus, flexible, Branch+Burn, NorthStar prototyping
- **<N> (Narrate)** = Clan — internal focus, flexible, shared context, CLAUDE.md lives here
- **C (Create)** = Hierarchy — internal focus, stable, governance, production standards
- **Market** (external focus, stable) = what INC *enables* — outside the design system's concern but the reason it exists

**Why:** Nate Baldwin's IDS 2026 talk "Prototyping for the Unknown" described Branch+Burn and structured prompting. Lincoln's key synthesis: his rule about not making code changes in early exploration stages is an I-protection mechanism — preventing premature C from contaminating I. The burn is Adhocracy logic, not discipline.

**Core claim:** The design system's most important layer is not the component library (C/Hierarchy) — it is the Clan/Narrate layer. CLAUDE.md is the Clan layer made machine-readable.
